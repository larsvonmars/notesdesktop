## Key Design Decisions

### 1. Custom Events for Click Handling
**Why**: Works in contenteditable context, survives serialization
**Alternative**: React event handlers (don't work with innerHTML)

### 2. Data Attributes for Persistence
**Why**: Preserved through DOMPurify, easy to parse
**Alternative**: JSON in a custom attribute (harder to query with CSS)

### 3. CustomBlockDescriptor Interface
**Why**: Extensible, reusable for other custom blocks
**Alternative**: Hard-coded in RichTextEditor (not scalable)

### 4. Dialog for Note Selection
**Why**: Better UX than autocomplete, shows folder context
**Alternative**: Wiki-style [[]] autocomplete (could be added later)

### 5. Event-Based Communication
**Why**: Decoupled components, easier to test
**Alternative**: Direct function calls (tighter coupling)
